# Design: JSON Schema & llms.txt Generator

Date: 2026-02-16
Status: Approved

## Purpose

SEO-helper tool for clients. Users generate JSON Schema (schema.org JSON-LD) and llms.txt/llms-full.txt files for their monitored domains. The tool scans existing crawl data, uses AI to generate contextually optimal outputs, and provides platform-specific implementation instructions with ready-to-paste snippets.

## Architecture: Single-Pass AI Generation

Uses existing OnSite crawl data (domainOnsitePages, schemaValidation, sitemap, robots, keywords) as input context. One AI call per generation. No live content fetching — relies on already-crawled metadata (titles, meta descriptions, headings h1-h4, word count, readability scores, URL structure).

Trade-off: slightly less deep than live-fetching full page content, but fast (15-45s), cheap (1 AI call), and leverages data we already have.

## Data Model

New table `generatorOutputs`:

```
generatorOutputs
  domainId        → ref("domains")
  type            → "jsonSchema" | "llmsTxt" | "llmsFullTxt"
  version         → number (auto-increment per domain+type)
  status          → "pending" | "generating" | "completed" | "failed"
  content         → optional string (generated output)
  metadata        → optional object {
                      pagesAnalyzed: number,
                      schemaTypes?: string[],     // for jsonSchema
                      sections?: string[],        // for llms
                      tokensUsed?: number,
                      generationTimeMs?: number
                    }
  createdAt       → number (timestamp)
  error           → optional string
```

Indexes: `by_domain_type` on `[domainId, type]`, `by_domain` on `[domainId]`.

History is preserved — each generation creates a new versioned record. Users can browse and compare previous generations.

## AI Generation Pipeline

### Generator 1: JSON Schema (schema.org JSON-LD)

Input data collected from existing tables:
- `domainOnsitePages` — URL, title, metaDescription, h1, htags (h1-h4), wordCount, readabilityScores (top 50 pages by pageScore)
- `schemaValidation` — existing schema.org types already on pages
- `domains` — businessDescription, targetCustomer
- `keywords` (top 20 by position) — currentUrl, keyword, currentPosition
- `domainSitemapData` — URL list from sitemap

AI generates:
- `WebSite` schema for homepage
- `Organization` or `LocalBusiness` based on businessDescription
- Per-page schema with auto-detected types: Article, Product, FAQ, HowTo, Service, Event, Recipe, etc. Detection based on URL path patterns, title, heading structure, word count
- `BreadcrumbList` from URL structure
- `SiteNavigationElement` if navigation pattern detected

Output: array of JSON-LD objects, each with `@context`, `@type`, and properties.

### Generator 2 & 3: llms.txt + llms-full.txt

Generated together in one AI call. Format follows llmstxt.org specification.

**llms.txt** (concise):
```markdown
# Site Name

> Short 1-2 sentence description.

## Docs
- [Page Title](URL): Brief description

## Blog
- [Article Title](URL): Brief description

## Products
- [Product](URL): Brief description

## Optional
- [Less important page](URL): Brief description
```

AI groups pages into logical H2 sections, writes short descriptions, determines what goes in "Optional".

**llms-full.txt** (expanded):
- Fuller descriptions (2-3 sentences per page)
- Additional sections: FAQ (AI-generated from content), Key Topics, Glossary
- Per-section summaries (text block under H2 before link list)
- "Optional" section included with full descriptions

AI returns JSON with `{ llmsTxt: "...", llmsFullTxt: "..." }`. Stored as two separate `generatorOutputs` records.

## Platform Integration Layer

After generating output, user selects target platform. System generates:

### For JSON Schema:

Platforms: Shoper, WordPress, WooCommerce, Shopify, Prestashop, Custom/HTML

Per platform:
- Ready-to-paste code snippet (e.g., `<script type="application/ld+json">` wrapped in platform-specific format)
- Step-by-step implementation instructions

Examples:
- Shoper: snippet for "Appearance > Template Editor > header" or "Custom HTML Codes"
- WordPress: functions.php snippet with `wp_head` hook, or plugin recommendation (Rank Math / Yoast)
- Shopify: snippet for `theme.liquid` in `<head>`
- Custom: raw `<script>` tag

### For llms.txt / llms-full.txt:

Platform-specific upload instructions:
- Shoper: FTP/SFTP upload to root directory
- WordPress: FTP upload or plugin-based
- Shopify: not directly possible in root — alternative via Cloudflare Worker / redirect rule
- Custom: upload to document root

Instructions generated by AI on-demand (not hardcoded) via `generatePlatformInstructions` action.

## Convex Backend Structure

### New files:

`convex/generators.ts` — queries + mutations:
- `getGeneratorOutputs(domainId, type?)` — fetch generation history
- `getLatestOutput(domainId, type)` — latest output per type
- `createGeneratorOutput(domainId, type, content, metadata)` — save new output
- `updateGeneratorStatus(outputId, status, content?, error?)` — update status

`convex/actions/generateSchema.ts` — action:
- Collects: domainOnsitePages (top 50), schemaValidation, domain info, top keywords, sitemap
- Builds AI prompt
- Calls `callAI()` via aiProvider
- Parses response, validates JSON-LD structure
- Saves to `generatorOutputs`

`convex/actions/generateLlmsTxt.ts` — action:
- Collects: all crawled pages, sitemap, domain info, keyword-to-page mapping
- Builds prompt (requests both llms.txt and llms-full.txt)
- Calls `callAI()`
- Parses JSON response with both formats
- Saves two records to `generatorOutputs`

`convex/actions/generatePlatformInstructions.ts` — action:
- Input: outputId + platform identifier
- Fetches content from generatorOutputs
- AI generates platform-specific instruction + snippet
- Returns directly (not persisted — lightweight on-demand)

## UI: "Generators" Tab

New tab on domain detail page. Two panels side-by-side (desktop) / stacked (mobile):

### Panel 1: JSON Schema Generator
- Header "JSON Schema (schema.org)"
- Status: "Not generated" / "Last generated: [date]"
- "Generate Schema" button (primary)
- After generation: JSON-LD preview with syntax highlighting, detected types list (Article x5, Product x3, FAQ x2...), platform selector dropdown, instruction + snippet display
- Actions: "Copy All", "Download .json", "Regenerate"
- History: list of previous generations with date and version, click to preview

### Panel 2: llms.txt Generator
- Header "llms.txt & llms-full.txt"
- Status: "Not generated" / "Last generated: [date]"
- "Generate" button (primary)
- After generation: two sub-tabs — "llms.txt" and "llms-full.txt", markdown preview, stats (pages included, sections, FAQ count)
- Platform selector → deployment instructions
- Actions: "Copy", "Download .txt", "Regenerate"
- History as above

### Generation Progress View
Skeleton/loader with steps:
1. "Collecting page data..."
2. "Analyzing content structure..."
3. "Generating with AI..."
4. "Finalizing output..."

Non-blocking — user can navigate away and return.
