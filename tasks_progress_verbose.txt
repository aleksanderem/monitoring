**Backend - Schema Updates (convex/schema.ts):**
Added serpFeatureTracking table:
- keywordId: v.id("keywords")
- date: v.string() // YYYY-MM-DD
- features: v.object() with 9 SERP features:
  * featuredSnippet (boolean)
  * peopleAlsoAsk (boolean)
  * imagePack (boolean)
  * videoPack (boolean)
  * localPack (boolean)
  * knowledgeGraph (boolean)
  * sitelinks (boolean)
  * topStories (boolean)
  * relatedSearches (boolean)
- fetchedAt: v.number()
- Indexes: by_keyword, by_keyword_date

**Backend - Queries Created (convex/serpFeatures_queries.ts):**
1. getSerpFeaturesByKeyword
   - Args: keywordId, days (optional, default 30)
   - Returns: Historical SERP features for keyword sorted by date
   - Filters by date range

2. getCurrentSerpFeatures
   - Args: keywordId
   - Returns: Most recent SERP features for keyword
   - Used for table badges display

3. getSerpFeaturesSummary
   - Args: domainId, days (optional, default 30)
   - Returns: Aggregated statistics across all keywords in domain
   - Calculates: featurePercentages, featureCounts, totalKeywords, totalDataPoints
   - Used for chart and overview section

4. getSerpFeaturesTimeline
   - Args: keywordId, days (optional, default 90)
   - Returns: Timeline of when features appeared/disappeared
   - Tracks changes between consecutive records
   - Used for historical analysis

**Backend - Mutations Created (convex/serpFeatures_mutations.ts):**
1. saveSerpFeatures
   - Args: keywordId, date, features object
   - Upserts SERP features record (updates if exists, inserts if new)
   - Updates fetchedAt timestamp
   - Returns: record _id

2. bulkSaveSerpFeatures
   - Args: array of {keywordId, date, features} records
   - Batch upsert for multiple keywords
   - Returns: {saved: count, ids: array}
   - Optimized for bulk imports from API
